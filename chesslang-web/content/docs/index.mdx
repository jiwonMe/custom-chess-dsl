## The Three Levels

ChessLang provides three levels of complexity to match your needs:

### Level 1: Configure

Simple YAML-like configuration for basic modifications.

```chesslang
game: "King of the Hill"
extends: "Standard Chess"

board:
  zones:
    hill: [d4, d5, e4, e5]

victory:
  add:
    hill: King in zone.hill
```

Perfect for:
- Adding victory/draw conditions
- Modifying board setup
- Defining zones
- Adjusting rules

[Learn Level 1 →](/docs/language/level1)

### Level 2: Compose

Declarative DSL for custom pieces and triggers.

```chesslang
piece Amazon {
  move: slide(orthogonal) | slide(diagonal) | leap(2, 1)
  capture: =move
  traits: [jump]
}

trigger explosion {
  on: capture
  do: remove pieces in radius(1) from target
}
```

Perfect for:
- Custom piece definitions
- Board effects
- Event triggers
- State tracking

[Learn Level 2 →](/docs/language/level2)

### Level 3: Script

JavaScript-like scripting for complex logic.

```chesslang
script {
  game.on("move", function(event) {
    if (event.piece.state.rage >= 3) {
      transform(event.piece, "RagingBerserker");
    }
  });
}
```

Perfect for:
- Complex AI behavior
- Dynamic abilities
- Custom game flow
- Advanced validation

[Learn Level 3 →](/docs/language/level3)

## Standard Chess Pieces

ChessLang inherits these pieces from "Standard Chess":

| Piece | Movement | Value |
|-------|----------|-------|
| King | `step(any)` | ∞ |
| Queen | `slide(any)` | 9 |
| Rook | `slide(orthogonal)` | 5 |
| Bishop | `slide(diagonal)` | 3 |
| Knight | `leap(2, 1)` | 3 |
| Pawn | `step(forward)` | 1 |
